ТЗ: Комментарии к позициям листа реализации (с правами по ролям)
Цель

Добавить к каждой позиции листа реализации блок комментариев, чтобы участники проекта могли оставлять сообщения любой длины. Удалять комментарии могут только Администратор и Прораб. Остальные роли могут добавлять, но не удалять.

Роли и права

Администратор

Добавлять комментарии к любой позиции.

Удалять любые комментарии.

Видеть все комментарии.

Прораб

Добавлять комментарии к любой позиции в своих проектах.

Удалять любые комментарии в своих проектах.

Видеть все комментарии в своих проектах.

Мастер

Добавлять комментарии к позициям своих проектов.

Не может удалять комментарии.

Видит все комментарии в своих проектах.

Заказчик

Добавлять комментарии к позициям своих проектов.

Не может удалять комментарии.

Видит комментарии, помеченные как «видно заказчику» (см. Видимость).

Доступ к спискам позиций и самим позициям остаётся по текущим правилам ролей.

Где появляются комментарии

В карточке каждой позиции листа реализации:

Заголовок: «Комментарии».

Счётчик комментариев (например, «Комментарии (3)»).

Список комментариев, отсортированный по времени (сверху новые или по кнопке «Сначала новые/старые»).

Поле ввода + кнопка «Добавить комментарий».

Структура комментария

Для каждого комментария сохранять и отображать:

Автор (имя + роль).

Время (дата и часы:минуты).

Текст (любой длины).

Видимость:

«Внутренний» (видят Админ/Прораб/Мастер),

«Для заказчика» (видят все роли, включая Заказчика).

Привязка к проекту и конкретной позиции.

Идентификатор комментария (для удаления и ссылок).

Статус (активен / удалён). При удалении в интерфейсе комментарий исчезает; в истории изменений фиксируется факт удаления.

По умолчанию при добавлении:

Комментарии от Заказчика всегда «Для заказчика».

Комментарии от Мастера/Прораба/Админа — по умолчанию «Внутренний», с переключателем «Показать заказчику».

Операции
Добавление комментария

Доступно: Администратор, Прораб, Мастер, Заказчик (в рамках доступных проектов/позиций).

Поля при добавлении:

Текст (обязателен).

Переключатель «Показать заказчику» (недоступен для Заказчика, у него всегда «Для заказчика»).

После добавления комментарий мгновенно появляется в списке, счётчик обновляется.

Удаление комментария

Доступно: Администратор (везде), Прораб (в своих проектах).

Действие необратимое для пользователя; запись об удалении фиксируется в истории изменений.

Для Мастера и Заказчика кнопка удаления не показывается.

Просмотр

Фильтры:

«Все» (для внутренних ролей),

«Для заказчика» (что увидит Заказчик),

По автору,

По периоду дат.

Переключатель сортировки: новые сверху / снизу.

Видимость (важно)

Внутренние комментарии не видны Заказчику.

Для заказчика видны всем, включая Заказчика.

Комментарии Заказчика — всегда «Для заказчика».

На каждой позиции добавить быстрый фильтр «Показать как видит клиент» (превью клиентской видимости).

Уведомления (минимум)

При добавлении комментария:

Внутренний → уведомить участников проекта (Админ, Прораб, Мастер) по внутренним уведомлениям.

«Для заказчика» → дополнительно уведомить Заказчика по внутренним уведомлениям (если у нас есть канал уведомлений; e-mail для Заказчика не обязателен в этом ТЗ).

Уведомление содержит: проект, позицию, автора, первые 120 символов текста, ссылку «Открыть позицию».

(Для Администратора e-mail-уведомления по фото/прогрессу у нас уже есть отдельным ТЗ; сюда e-mail не добавляем, чтобы не дублировать.)

История изменений

Каждое действие записывать:

Добавлен комментарий: кто, когда, к какой позиции, видимость («внутренний»/«для заказчика»).

Удалён комментарий: кто, когда, ссылка на изначальные метаданные (текст в историю не выводим целиком, достаточно ссылки/ID, чтобы не раскрывать лишнего).

Изменения видимости (если предусмотрите в будущем): кто, когда, что стало.

Просмотр истории:

В карточке позиции — мини-лента «История» (включая события по комментариям).

В общей истории проекта — события по всем позициям.

Ограничения и защита

Текст — любой длины. Для устойчивости интерфейса:

Автоперенос строк,

Обрезка в списке (например, превью 3–5 строк) с кнопкой «Показать полностью».

Анти-спам: не более N комментариев за 30 секунд с одного пользователя (значение N задать в настройках).

Заказчик не может видеть/удалять внутренние комментарии и не видит переключатель видимости.

При ошибке сохранения — понятное сообщение и повтор без потери текста (оставить текст в поле).

Пагинация комментариев по 20–50 штук с догрузкой по кнопке «Ещё».

Интерфейс (UX)

В каждой позиции:

Заголовок «Комментарии (X)».

Поле ввода текста + кнопка «Добавить».

Переключатель «Показать заказчику» (только для Админа/Прораба/Мастера; для Мастера доступен, но по умолчанию «внутренний»).

Список комментариев:

Аватар/инициалы автора (если есть), имя и роль, время.

Текст (превью с «Показать полностью»).

Бейдж видимости: «Внутренний» / «Для заказчика».

Кнопка «Удалить» — видна только Админу/Прорабу по правилам.

Быстрые фильтры: «Все / Для заказчика / Мои».

Приёмочные тесты

Добавление: все роли успешно добавляют комментарии на доступных им позициях; Заказчик — только «для заказчика».

Удаление: только Админ и Прораб могут удалить; после удаления у других ролей комментарий исчезает.

Видимость: внутренние комментарии не видны Заказчику; «для заказчика» видны всем.

Фильтры/сортировка: работают по виду видимости/автору/дате; переключение «новые сверху/снизу».

История изменений: события добавления/удаления фиксируются корректно.

Стабильность: длинные тексты отображаются аккуратно; превью раскрывается; пагинация и «Ещё» работают.

Уведомления: участники проекта получают уведомления по правилам.

Права: у Мастера и Заказчика нет кнопки удаления; у Прораба/Админа — есть, согласно проектной принадлежности.

Регресс: существующие функции листа реализации (прогресс, фото) не нарушены.

Чек-лист для студентов

 Блок «Комментарии» в карточке каждой позиции.

 Добавление комментариев: все роли, любой длины, с видимостью.

 Удаление комментариев: только Админ/Прораб (по проектам).

 Видимость: «Внутренний» / «Для заказчика», дефолты по ролям.

 Фильтры и сортировка, счётчик X.

 Пагинация/дозагрузка, превью длинных текстов.

 Уведомления о новых комментариях.

 История изменений (добавлен/удалён).

 Сообщения об ошибках, сохранение текста при сбое.

 Регресс-проверка: ничего не сломано в листе реализации.
