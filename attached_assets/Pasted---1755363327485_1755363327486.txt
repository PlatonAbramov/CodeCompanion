Задание: «Смета → Лист реализации» с прогрессом и медиа
Цель

После загрузки сметы в проект (файл будет называться в стиле “invoice …”) система автоматически превращает её в лист реализации:

строки-позиции работ,

прогресс по каждой позиции в процентах,

чекбокс «Выполнено полностью»,

миниатюры фото по каждой позиции,

по клику на миниатюру — модалка с большой фотографией, стрелки вперёд/назад, закрытие.

Входные данные

Файл сметы, загруженный в карточку проекта. Имя файла начинается с “invoice”.

Поддерживаемые форматы для старта: PDF, XLSX/CSV, изображение (как минимум PDF и XLSX; если изображение — распознать текст).

Смета содержит: позиции работ, единицы/количество (если есть), цену/сумму (если есть), примечания (если есть).

Преобразование в «Лист реализации»

После успешной загрузки “invoice …” система предлагает «Создать лист реализации из сметы».

Парсинг сметы в набор строк-позиций. Для каждой строки создать запись с полями:

Наименование позиции (обязательно).

Количество/единица (если есть в смете).

Стоимость/сметная сумма (если есть).

Примечание/описание (если есть).

Прогресс, % (0–100).

Флаг “Выполнено полностью” (да/нет).

Список прикреплённых фото (0..N).

Дата последнего обновления и автор обновления.

Если парсинг неоднозначен, показываем предпросмотр таблицы и даём подтвердить/исправить названия столбцов.

Лист реализации — интерфейс в проекте

Раздел внутри проекта: «Лист реализации».

Список позиций (постранично при больших объёмах).

В каждой строке видно:

Наименование позиции.

Прогресс % + шкала.

Чекбокс «Выполнено полностью».

Кнопка/зона «Фото» с миниатюрами (если есть).

Иконка «Добавить фото».

Дата и автор последнего изменения.

Управление прогрессом и статусом

Прогресс редактируется числом (0–100) или ползунком.

Чекбокс «Выполнено полностью»:

При установке ставит прогресс = 100%.

При снятии оставляет прогресс без изменений (можно вручную менять).

При любом изменении — сохраняем, фиксируем автора и время.

Фото к позиции

У каждой позиции можно прикрепить 1..N фотографий (из файлов).

Показ миниатюр (маленькие превью) прямо в строке.

Клик по миниатюре открывает модалку:

Большое изображение по центру.

Стрелки вперёд/назад для листания фото этой же позиции.

Номер текущего фото и общее количество (например, 2/5).

Крестик/кнопка Закрыть.

Клик вне области — закрывает модалку.

Поддержать базовую ориентацию/масштаб (чтобы вертикальные фото смотрелись корректно).

Ограничить размер загружаемых файлов и количество фото на позицию (настройка).

Расчёт общего прогресса проекта (в этом разделе)

Показать сверху агрегированный прогресс по листу реализации:

По умолчанию — среднее арифметическое прогрессов всех позиций.

Если у позиции есть количество или сумма — предусмотреть опцию «взвешивать» прогресс по количеству/сумме (в будущем).

Отдельно вывести количество выполненных полностью / общее число позиций.

Права доступа

Администратор: полное управление листом реализации (создание из сметы, редактирование строк, прогресс, фото, удаление).

Прораб: редактирование прогресса и статусов по позициям своих проектов, добавление фото, удаление только своих фото.

Мастер: редактирование прогресса только по назначенным позициям/своим проектам, добавление фото, удаление только своих фото.

Заказчик: только просмотр листа реализации и фото (только тех, что помечены «показывать заказчику» — см. ниже).

Метки видимости для фото/позиции

При добавлении фото — чекбокс «показывать заказчику».

По умолчанию фото доступно только внутренней команде.

Для позиции можно задать флаг «позиция видна заказчику» (если выключено — не показываем позицию в кабинете заказчика).

Уведомления (минимум)

Изменение прогресса позиции → уведомить участников проекта (Прораб/Админ).

Установка «Выполнено полностью» → уведомить участников проекта.

Добавление фото → уведомить Прораба/Админа (опционально).

Журнал изменений (аудит)

По каждой позиции хранить историю:

Что изменено (прогресс, статус).

Кто изменил.

Когда изменил.

По фото: кто добавил/удалил, когда.

Ошибки и защита от потерь данных

При неудачном парсинге сметы — понятное сообщение, предложить ручное сопоставление колонок.

При попытке выйти со страницы с несохранёнными изменениями — предупреждать.

При загрузке неподдерживаемого формата — сообщить допустимые форматы и размер.

Производительность и удобство

Ленивая подгрузка фото в списке (миниатюры).

Модалка грузит только соседние фото (текущее, предыдущее, следующее).

Поиск/фильтр позиций по названию и статусу (все/в процессе/выполнены).

Массовые операции:

Выбрать несколько позиций → «отметить выполнено» / «установить прогресс N%».

Массовая загрузка фото к выбранной позиции (пакетом).

Приёмка (что проверить)

Загрузили “invoice …” → система предложила создать лист реализации → позиции корректно распознаны.

Можно вручную поправить соответствие колонок перед созданием.

В листе видно наименование, прогресс, чекбокс, фото-миниатюры.

Чекбокс «Выполнено полностью» ставит 100% и сохраняет.

Миниатюра открывает модалку, стрелки листают, закрытие работает.

Добавление нескольких фото к одной позиции — все видны и листаются.

Общий прогресс рассчитывается корректно.

Права соблюдаются: Админ всё, Прораб и Мастер — по своим, Заказчик — только просмотр видимых элементов.

Логируются изменения.

Большие файлы/неподдерживаемые форматы — корректные ошибки.

Не включаем сейчас (можно потом)

Импорт нескольких смет в один лист с слиянием.

Взвешенный прогресс по сумме/количеству — как настройка по умолчанию.

Комментарии к каждой фотографии.

Версионирование листа реализации.

Чек-лист для выполнения

 Кнопка «Создать лист реализации» при загрузке “invoice …”.

 Парсер сметы → предпросмотр → подтверждение → создание позиций.

 Лист реализации в проекте: список позиций с прогрессом и флагом «выполнено».

 Прикрепление фото к позиции, миниатюры в строке.

 Модалка фото: большое изображение, стрелки, номер/из N, закрыть.

 Права по ролям (см. выше).

 Метка «показывать заказчику» для фото и позиции.

 Журнал изменений.

 Уведомления по ключевым событиям.

 Тесты приёмки (10 пунктов из раздела выше).
