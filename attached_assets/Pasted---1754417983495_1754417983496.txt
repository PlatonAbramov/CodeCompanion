Техническое задание
📱 Название проекта: Мобильное приложение для строительной компании и сервисной команды по кондиционерам
1. 📌 Общая информация
Тип приложения: Мобильное (iOS / Android)

Назначение: Для внутреннего использования сотрудниками (директорами, инженерами, мастерами)

Языки интерфейса: Русский, Английский

Интеграции: amoCRM и другие CRM не нужны

Авторизация через соцсети: Не требуется

Чат/переписка между пользователями: Не нужен

2. 👤 Пользователи и роли
2.1. Директор (Admin)
Полный доступ ко всем проектам и функциям

Создание нового проекта

Указание общей стоимости проекта

Добавление/редактирование авансов

Загрузка смет, разрешений, документации (PDF/фото)

Добавление сотрудников: выдача логина и пароля

Просмотр всех расходов по проектам

Возможность редактирования любых данных

Просмотр Сводной информации (Доход – Аванс – Расходы – Прибыль)

2.2. Мастер / Ремонтник (User)
Входит в приложение по логину и паролю, выданному директором

Имеет доступ только к своим расходам

Может:

Добавить расход (с указанием суммы, категории, описания)

Обязательно прикрепить фото/файл чека

Не может редактировать чужие данные

Не видит финансовой сводки проекта

3. 📂 Структура приложения
3.1. Экран проектов
Список всех проектов

Кнопка Создать проект (для директора)

В каждом проекте:

Название проекта

Общая стоимость

Сумма аванса (можно редактировать и добавлять частями)

Сроки реализации

Смета (PDF или фото)

Рабочая документация

Кнопка Добавить расход

Кнопка Сводная информация

3.2. Сводная информация (по каждому проекту)
Доход = Общая стоимость проекта

Аванс = Все внесенные авансы

Расходы = Сумма всех расходов, добавленных мастерами

Прибыль = Доход – Расход

Всё отображается в табличной или карточной форме

3.3. Экран «Добавить расход»
Категория расхода (на выбор или вручную)

Сумма

Комментарий (по желанию)

Фото/файл чека (обязательно)

Кнопка Отправить

Автоматическая запись, кто добавил и когда

3.4. Управление сотрудниками (только для директора)
Список сотрудников

Добавить нового сотрудника

Выдача логина и пароля

Удаление или блокировка сотрудника

4. 🛠️ Дополнительные требования
Интуитивно понятный интерфейс (в духе Uber / Notion / Trello)

Оффлайн-режим с последующей синхронизацией при подключении к интернету (по возможности)

Возможность экспорта данных (по проекту) в PDF (по кнопке)

Доступ к данным ограничивается по ролям

Данные хранятся безопасно, с возможностью резервного копирования
