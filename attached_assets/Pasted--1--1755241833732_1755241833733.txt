ТЗ: Отображение фотографий в истории движений инструмента
1. Цель

При выдаче или возврате инструмента пользователь может прикрепить фотографию.
В карточке инструмента в разделе История движений должна отображаться миниатюра (thumbnail) прикреплённой фотографии, а при клике на неё — фото должно раскрываться в увеличенном виде без открытия нового окна.

2. Функциональные требования
2.1 Прикрепление фото

Возможность загрузить фото как при действии Выдача, так и при действии Возврат.

Форматы: JPG, PNG, WEBP.

Максимальный размер файла: 5 МБ.

При загрузке автоматически создавать уменьшенную версию (thumbnail) для показа в истории.

2.2 Отображение в истории движений

Для каждой записи в истории (Выдано / Возвращено), если есть фото:

Показывать миниатюру (100×100 px или меньше).

Расположение миниатюры — слева от текста события или рядом с кнопкой удаления.

При наведении курсора — курсор-лупа (pointer), лёгкое увеличение в hover.

2.3 Просмотр в увеличенном виде

При клике на миниатюру открывается увеличенное фото:

Открытие внутри той же вкладки (модальное окно поверх карточки).

Модальное окно затемняет фон.

Кнопка «Закрыть» (X) в правом верхнем углу модалки.

Возможность закрыть кликом по фону или клавишей Esc.

Фото в модалке отображается с сохранением пропорций, максимум по ширине/высоте экрана.

Поддержка зума (опционально).

3. Нефункциональные требования

Миниатюры хранить отдельно от оригиналов (для скорости загрузки списка).

Оптимизировать фото на сервере при загрузке (сжатие без сильной потери качества).

Для мобильных устройств:

Миниатюры уменьшить до ~60 px.

Модальное фото — адаптивное, по ширине экрана.

Вся функциональность должна работать без перезагрузки страницы (AJAX / SPA).

4. API и данные
4.1 Хранение

В БД для записи истории хранить:

Путь к оригиналу фото.

Путь к миниатюре.

Если фото нет — поле пустое.

4.2 API

При получении истории движений API возвращает для каждой записи:
{
  "action": "Выдано",
  "user": "Эльдар",
  "phone": "+971543959595",
  "date": "2025-08-12T21:49:00",
  "photo_thumbnail_url": "https://.../thumb.jpg",
  "photo_original_url": "https://.../original.jpg"
}
5. UX/UI

Миниатюры не должны ломать текущий дизайн истории.

Если фото нет — отображается как сейчас (только текст события).

Переход на модальное окно и обратно — плавная анимация (fade-in / fade-out).

Модальное окно адаптировано под тёмную/светлую тему (если есть в проекте).

6. Приёмка

Загрузить фото при выдаче и возврате → фото видно в истории.

Миниатюра кликабельна, открывается модалка без перезагрузки страницы.

На мобильных всё масштабируется корректно.

Если фото нет — история отображается без изменений.
