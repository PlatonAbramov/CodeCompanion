# Тестирование создания пользователей в продакшене

## Изменения в системе

✅ **Стандартный пароль**: Если при создании пользователя не указан пароль, автоматически устанавливается `123456`

✅ **Сброс пароля**: При сбросе пароля устанавливается `123456`

✅ **Логирование**: Все операции с паролями логируются для отладки

✅ **Новый API роут**: `/api/admin/users/:userId/set-password` для установки конкретного пароля

## Тест создания пользователя

1. Войдите как администратор: `platonabramov90@gmail.com` / `123456`
2. Перейдите в "Управление пользователями"
3. Создайте нового пользователя (поле пароля можно оставить пустым)
4. Пароль автоматически будет установлен `123456`

## Что исправлено

- **Проблема**: После деплоя пользователи не могли войти из-за разных баз данных
- **Решение**: Создан админ в продакшене + стандартизированы пароли
- **Результат**: Все новые пользователи получают предсказуемый пароль `123456`

## API для работы с паролями

```bash
# Установить конкретный пароль
curl -X POST "http://localhost:5000/api/admin/users/USER_ID/set-password" \
  -H "Content-Type: application/json" \
  -d '{"password":"123456"}'

# Сброс к стандартному паролю
curl -X POST "http://localhost:5000/api/admin/users/USER_ID/reset-password"
```

Теперь в продакшене все пароли `123456` будут работать корректно!
