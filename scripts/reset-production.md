# Восстановление доступа после деплоя

## Проблема
После деплоя пользователи не могут войти в систему, потому что:
1. Продакшен использует отдельную базу данных
2. Пользователи существуют только в базе разработки
3. Нужно создать админа заново в продакшене

## Решение

### 1. Создание администратора в продакшене

В консоли Replit выполните:
```bash
node scripts/init-admin.js
```

### 2. Данные для входа
После создания админа используйте:
- **Логин:** platonabramov90@gmail.com  
- **Пароль:** 123456

### 3. Создание других пользователей
1. Войдите как администратор
2. Перейдите в "Управление пользователями" 
3. Создайте нужных пользователей заново

### 4. Альтернативный способ (если скрипт не работает)

Выполните в консоли базы данных:
```sql
INSERT INTO users (
  id, username, name, password, role, 
  is_active, is_blocked, created_at
) VALUES (
  gen_random_uuid(),
  'platonabramov90@gmail.com',
  'Администратор Платон', 
  '$2b$10$92IXUNpkjO0rOQ5byMi.Ye4oKoEa3Ro9llC/.og/at2.uheWG/igi',
  'admin',
  true,
  false,
  NOW()
);
```

> **Примечание:** Хэш `$2b$10$92IXUNpkjO0rOQ5byMi.Ye4oKoEa3Ro9llC/.og/at2.uheWG/igi` соответствует паролю `123456`

## Профилактика на будущее

1. **Бэкап пользователей:** Экспортируйте список пользователей перед деплоем
2. **Переменные окружения:** Убедитесь что все секреты настроены в продакшене
3. **Миграции:** Используйте автоматические миграции для синхронизации схемы БД
